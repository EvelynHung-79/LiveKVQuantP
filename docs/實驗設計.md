## **預期驗證實驗 (Expected Validation Experiments)**

### **A. 主要效能評估 (Main Comparative Experiments)**

- **目標**：證明 LiveKVQuantP 在「顯存-精度-速度」的綜合權衡上優於現有方法。
- **比較對象**：
    - **FullKV**（Upper Bound）
    - **Finch**（prefill + top-r% token selection，主 baseline）
    - **KVQuant**（decoding-phase quantization，support baseline）
- **評估指標**：
    1. **F1 Score / Accuracy**（LongBench 任務）
    2. **Memory Peak (GB)**
    3. **End-to-End Latency**（拆分 Prefill / Decode）

> ⚠️ 注意：
> 
> - Finch 會丟掉部分 token，你方法不丟 token → narrative 可強調「representation compression vs token dropping」
> - KVQuant 主要優化 decoding → 拆解 prefill / decode latency，避免不公平比較
> - 壓縮率需對齊（例如相同 bit width / memory ratio）

### **B. 消融實驗 (Ablation Studies) (部分取樣的理由？**

### **層次一：架構驗證 (Architectural Verification)**

- **Warmup Chunk Strategy**：比較首個區塊不量化 vs 全量化對長文本生成的影響
- **Hybrid Precision Strategy**：全層量化 vs 前 $N$ 層保留 FP16
- **Outlier Isolation**：純 Dense 量化 vs Dense + Sparse 分離儲存
- **Statistics Smoothing**：EMA Absmax vs EMA MinMax 更新策略對量化穩定性影響

### **層次二：參數敏感度 (Sensitivity Analysis)**

- **Outlier Ratio**：不同稀疏比例 (0.5% / 1.0%) 對 F1 Score 與 Memory Peak 的影響
- **Chunk Size Impact**：不同 Chunk Size (512 vs 4096) 對 Peak Memory 與 System Latency 的影響
